Problem interpretation and causality

At every annotated pause the system must decide whether the user is finished. A false positive during a hold pause can interrupt the user; a false negative at the true end forces the agent toward the 1.6-second timeout. The scorer sweeps a probability threshold and an action delay, then selects the lowest mean end-of-turn response delay that interrupts no more than 5% of turns.

The implementation makes causality explicit at the start of every feature call:

end = int(round(pause_start * sr))
prefix = x[:end].copy()
analysis = _analyse_prefix(prefix, sr)

Every subsequent frame, spectrum, energy statistic and pitch estimate is computed from prefix. The inference path does not read pause_end, label, the current pause duration, the file duration, or whether the current row is the final annotation. The full WAV may be cached for I/O efficiency, but no sample at or after pause_start reaches the feature extractor.

Model and training strategy

The saved model averages three small classifiers: a sparse L1 logistic regression, Extra Trees and histogram gradient boosting. Linear and nonlinear components are combined because the data set is small: the linear model stabilizes broad trends, while the tree models capture interactions such as a falling energy contour being useful only when the final voiced segment is sufficiently long.

Long hold pauses receive larger training weights. This is not an inference feature: the duration is used only while fitting on labelled training data, because long holds are the examples that can cause false cutoffs at practical action delays. The final artifact contains separate English and Hindi ensembles plus a pooled fallback for an unseen folder whose language cannot be recognized from the folder or turn IDs.

Validation uses five folds grouped by turn_id. No pause from a held-out turn appears in that fold's training set. Model choice was based on the official delay metric rather than accuracy alone.

Error analysis

The grouped predictions show two recurring failure types. First, a continuation can sound complete: a long pause after falling pitch and energy receives a high EOT score. Second, true endings with weak, clipped or highly variable final speech receive low scores because their final contours resemble hesitation. The most useful quick manual checks are listed in ERRORS_TO_REVIEW.md; examples include the 3.0-second English hold in en__091 and the Hindi EOT in hi__040.

The Hindi baseline is already strong on the supplied folder because relatively few hold pauses survive beyond roughly 0.85 seconds. The model's gain therefore comes from identifying enough true endings at a shorter action delay while suppressing the small set of long holds.

Why this beats the status quo

  • A silence-only endpoint treats every pause alike and must wait long enough to survive the longest hesitations. This model adds phrase-final acoustic evidence and a causal turn hazard.
  • The grouped English result reduces mean delay by 350 ms at the same interruption budget, while the grouped Hindi result improves an already favorable silence operating point.
  • The inference code is deployable: one command, deterministic CPU execution, a portable saved model, no test-label fitting, no internet access and no pretrained representation. It does not unpickle scikit-learn estimator objects.
  • Training is metric-aware: dangerous long holds matter more than short holds that end before the agent's action timer can fire.

Limitations and next work

The data set is small, and the supplied-folder 100 ms scores demonstrate fit rather than generalization. Language recognition currently uses the folder name or the conventional en__/hi__ turn prefix, with a pooled fallback when neither is available. With one more day, the highest-value work would be manual inspection of grouped errors, speaker-normalized final-syllable lengthening, nested grouped tuning of the scorer's low-false-cutoff region, and a compact sequence model trained from scratch on causal log-spectrogram patches.
